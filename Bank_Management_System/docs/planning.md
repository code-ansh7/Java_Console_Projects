# 🎯 Project Goal

To build a real-world console-based banking application using **Core Java** while learning Object-Oriented Programming, File Handling, Validation, and Software Design.

This project focuses on writing clean, modular, and maintainable code rather than building a complex banking software.

# 🧠 Design Decisions

## Why File Handling?

Instead of using MySQL, a text file was used to understand persistent data storage using Core Java.

---

## Why OOP?

To understand:

- Classes
- Objects
- Constructors
- Methods
- Object Passing
- Code Reusability

---

## Why Validation Class?

To separate validation logic from business logic.

---

## Why TransactionManager?

To keep transaction-related code separate from UI and file operations.

---

## Why FileManager?

To keep all file-related operations in a single class.
